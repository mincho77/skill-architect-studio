# Arquitectura de marca y Cobranding — Sistema de Diseño SURA

Fuente: Manual de marca, sección "Arquitectura de marca y cobranding".

---

## 1. Arquitectura de marca

SURA se organiza para ser **clara para las personas** y establecer la
relación entre la marca corporativa, sus subholdings y sus negocios.

Atributos no negociables:
- **Urgencia de claridad** en el propósito del producto
- **Brevedad, precisión, concisión**
- **Coherencia y recordación**

La secuencia, escritura, tipografía, color y ubicación que plantea esta
arquitectura **no es negociable**.

---

## 2. Nomenclatura de portafolio

> **Regla 80 % Monolítica:** la manera en que nombramos el portafolio SURA
> es estratégica y debe ser **sostenible en el largo plazo**. No se debe
> confundir el concepto de **campaña comercial** (estacional) con el
> **nombre del producto** (permanente).

### 2.1 Niveles
| Nivel | Definición |
|---|---|
| **Producto** | Conjunto de coberturas de financiación y/o gestión. Genérico, apela a la construcción cultural. Se radica en superintendencia y tiene nota técnica |
| **Plan** | Manera de presentar las condiciones específicas del producto. Oferta comercial que agrupa coberturas/servicios. **Solo se conecta a la marca de manera verbal, no visual** |
| **Categoría** | Tipo de negocio asegurador (ej. Autos) |
| **Servicio (Programa)** | Servicios se presentan siempre como **programas**. Para servicios se sugiere la **mayor claridad posible**, sin nombres conceptuales o poéticos |

### 2.2 Canales

| Canal | Característica |
|---|---|
| **Tradicional** | Asesores/agentes requieren claridad sobre el producto |
| **Digital** | El producto debe ser **muy sugestivo** en su nombre — no hay asesor que explique |

Ejemplo: `Seguro de salud → Plan de Salud Global`

## 3. Cobranding

### 3.1 Definición

Iniciativas en donde **dos o más compañías o marcas van juntas** y llegan a
un público específico. Funciona como un **matrimonio**: todos los actores
deben sentirse identificados con la comunicación.

### 3.2 Principios compartidos (no negociables)

Antes de usar cobranding, valida que ambas partes comparten:

1. **Transparencia**
2. **Responsabilidad**
3. **Equidad**
4. **Respeto**

Además debe cumplirse:
- Llegan a los **mismos públicos**
- Capitalizan los **canales de ambas compañías**
- Comunican con claridad los productos
- Todos los actores se sienten **identificados** con lo que van a hacer

### 3.3 Construcción visual

La figura de cobranding es la **suma de dos o más marcas separadas por una
línea delgada e imaginaria** que tiene de distancia al logo **1 cóndor
imaginario** (ver `logo.md` — el cóndor es el símbolo SURA).

- Permite mayor **legibilidad** de los logosímbolos.
- **No cierra ni respalda** la comunicación — la **abre**.
- Se ubica en la parte **superior o inferior** de la pieza.

---

## 4. Termómetro de cobranding

Indica el **porcentaje de participación de cada marca**. A mayor alianza,
mayor control de SURA sobre la comunicación: color azul, tipografía,
fotografías, ilustraciones, diagramación.

| Nivel | SURA | Aliado | Control visual |
|---|---|---|---|
| **L1** | 1 % | 99 % | Solo logo SURA presente — toda la comunicación es del aliado |
| **L2** | 20 % | 80 % | Logo SURA visible, paleta y tipografía mayoritariamente del aliado |
| **L3** | 50 % | 50 % | Equilibrio — convivencia de ambos sistemas |
| **L4** | 80 % | 20 % | SURA domina: azul, tipografía Sura, fotografías SURA. Aliado aparece secundario |
| **L5** | 100 % | 0 % | Comunicación totalmente SURA — el aliado solo está nombrado |

### 4.1 Qué cambia según el nivel

| Elemento | L1 (1 % SURA) | L3 (50/50) | L5 (100 % SURA) |
|---|---|---|---|
| Logo SURA | Sí, mínimo | Sí, equivalente | Sí, principal |
| Color SURA (Azul Cielo Latino) | No | Parcial | Sí, ≥40 % |
| Tipografía Sura | No | Compartida | Sí |
| Botones SURA | No | No | Sí |
| Contenido SURA (ilustración, fotografía, pictogramas, tipografía) | No | Parcial | Sí |
| Controladores SURA (radio, checkbox, switch) | No | No | Sí |
| Tags, campos de texto, modales, stepper, paginador | No | No | Sí |
| **Sistema de diseño SURA aplicado** | No | Parcial | **Completo** |

## 5. Usos correctos (Web)

| Sección | Regla |
|---|---|
| **Header** | Ubica los logotipos manteniendo proporciones y espaciados — alineados a la misma línea base |
| **Navegación** | Una sola fila de opciones, ambos logos visibles |
| **Hero** | Mensaje único, no mezclar dos titulares de marcas distintas |
| **Footer** | Coexistencia ordenada de columnas — cada marca conserva sus datos legales |
| **Componentes** | Cada componente sigue los lineamientos de **su marca respectiva**: botones SURA = sistema SURA, botones del aliado = sistema del aliado |

---

## 6. Usos incorrectos

**Nunca:**

1. **Redimensionar** los logotipos para que parezcan desiguales en tamaño/peso.
2. **Superponer** los logos con otros elementos.
3. Usar formatos que hagan parecer los logos desiguales.
4. **Mezclar componentes** de ambas marcas en una sola pieza (ej. botón con
   color de aliado pero tipografía Sura).
5. **Mezclar los colores** de las marcas de manera aleatoria.
6. Usar **fuentes que no estén especificadas** en los lineamientos de cada
   marca.
7. **Intervenir componentes** para combinarlos con elementos de la otra marca.

## 7. Plataforma de marca

Para aliados, SURA dispone una plataforma con piezas previamente
desarrolladas:

**`marca.sura.com`**

- Filtro disponible en la parte superior derecha: **Aliados**.
- Permite filtrar por tipo de tema.
- Los aliados pueden construir comunicaciones de manera ágil sin desviarse
  del sistema.

## 8. Reglas operativas para el skill

1. Cuando el mockup sea cobranding, **pregunta el nivel del termómetro**
   (L1 / L2 / L3 / L4 / L5) antes de generar.
2. En **L4–L5**, aplica el sistema SURA completo (colores, tipografía,
   componentes, ilustraciones, fotografías).
3. En **L1–L2**, el aliado lidera — limita la presencia de SURA al logo y
   la línea legal. **No** apliques color Azul Cielo Latino ni tipografía
   Sura más allá del logo.
4. En **L3 (50/50)**, divide el contenido por bloques: bloques SURA con
   sistema SURA, bloques aliado con sistema aliado. Nunca componentes
   híbridos.
5. **Separación entre logos**: usa la distancia de **1 ancho de símbolo
   SURA** (cóndor). El valor canónico es **26 px de ancho cuando el logo se
   renderiza a 32 px de alto** (ver `logo.md` §7 — el SVG real del símbolo mide
   26 px). La separación escala con el tamaño del logo:
   - Logo SURA a 32 px de alto → símbolo = 26 px → separación ≈ **26 px**.
   - Logo a otra altura H → separación ≈ `26 px × (H / 32)`.
   > ⚠ No usar el 18.69 px del mínimo absoluto del manual (`logo.md` §1) como
   > separación: ese es el ancho mínimo de impresión del isotipo, no el ancho
   > del cóndor a tamaño de pantalla. Para separación de cobranding manda el
   > **26 px** del SVG real (directiva en `logo.md` §7).
6. **No "intervengas" un componente** para combinarlo con elementos del
   otro sistema. Si necesitas un componente híbrido, escala todo el bloque
   al nivel del termómetro correspondiente.
7. Cuando el mockup sea **solo SURA (sin aliado)**, no apliques las reglas
   de cobranding — usa el sistema SURA puro.
8. Si el aliado no tiene marca propia, puede usar **solo su nombre** en
   tipografía neutra del sistema SURA.
